# Writing Great Skills

Reference for writing and editing skills well.

## Principles

1. **Name clearly** — the name should describe what the skill does in 2-3 words.
2. **Description is critical** — skills without good descriptions are never surfaced. Cover what it does AND when to trigger it.
3. **Frontmatter** — `name` (lowercase hyphen-separated), `description`, optional `disable-model-invocation`.
4. **Body** — the actual instructions. Be specific about the process, not vague.

## Structure

- Use headings for major phases
- Number steps when order matters
- Use bullet lists for options within a step
- Reference other skills by name with backticks

## Anti-patterns

- Vague descriptions ("helps with code")
- Missing the "when to trigger" part
- Too many responsibilities in one skill
- Not referencing related skills
